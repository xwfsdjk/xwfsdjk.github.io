git入门

12月 13 2017

四个区

  1. 工作区
  2. 暂存区
  3. 本地仓库
  4. 远程仓库

五种状态

  1. 未修改
  2. 已修改
  3. 已暂存
  4. 已提交
  5. 已推送

正常工作流程

  1. git add .
  2. git commit -m “描述”
  3. git push -u origin master 第一次提交 git push 提交


远程仓库

  1. git init
  2. git remote add origin https://github.com/xxxxxx
  3. blog-react git:(master) git remote -v 查看关联的远程仓库
  4. git remote set-url origin 修改远程仓库的地址
  5. git push -f orgin master # 强制覆盖远程分支
  6. git push origin :[要删除的远程分支名字]
  7. git remote rm origin git remote add origin git@gitee.com:liaoxuefeng/learngit.git 重新关联仓库

切换分支

  1. git checkout -b 新建分支并切换
  2. git branch 查看当前有哪些分支
  3. git branch -a 查看当前和远程的分支
  4. git checkout master 切换到master分支
  5. git merge dev 合并dev分支到当前分支
  6. git pull origin 拉取远程最新代码
  7. 本地有修改 git stash git pull git stash pop

撤销操作

  1. git checkout . 撤销工作区的所有修改
  2. git reset –hard 重置暂存区与工作区，与上一次commit保持一致
  3. git reset HEAD file 撤销某个文件暂存区的修改放到工作区

版本回退

  1. git log –graph –decorate –abbrev-commit –all 查看历史版本
  2. git checkout a5d88ea 切换到任意版本
  3. git push -f origin master 老版本覆盖远程仓库
  4. git reset –hard 回退到某个版本
  5. git reflog 回退反悔了

配置git

  1. git config –list
  2. git config –global user.name ““
  3. git config –global user.email ““
  4. git config –global alias.co checkout
  5. git config –global alias.ci commit
  6. git config –global alias.br branch

bug修复

  1. 在 dev 下正常开发中，说有1个bug要解决，首先我需要把dev分支封存stash
  2. 在master下新建一个issue-101分支，解决bug，成功后
  3. 在master下合并issue-101
  4. 在 dev 下合并master， 这样才同步了里面的bug解决方案
  5. 解开dev封印stash pop，系统自动合并 & 提示有冲突，因为封存前dev写了东西，此时去文件里手动改冲突
  6. 继续开发dev，最后add，commit
  7. 在master下合并最后完成的dev

标签

  1. git tag v1.0 打标签
  2. git tag 查看所有标签
  3. git tag v0.9 6224937 git log –pretty=oneline –abbrev-commit 某个commit 打标签
  4. git show v0.9 查看某个标签
  5. git tag -d v0.1 删除标签
  6. git push origin v1.0 推送标签
  7. git push origin –tags 一次性推送全部尚未推送到远程的本地标签
